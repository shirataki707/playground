# 効果的なスキルのチェックリスト

スキルをレビューする際に使用する完全なチェックリスト。

---

## コア品質

- [ ] `name` フィールドが具体的で**動名詞形**（動詞 + -ing）を使用している（名詞句・アクション形は不可）
- [ ] `name` が曖昧・過度に一般的でない（`helper`、`utils`、`data`、`files` 等でない）
- [ ] `description` がスキルが何をするかを具体的に説明している
- [ ] `description` がいつ使用するかを具体的に説明している
- [ ] `description` が三人称で書かれている（「〜できます」「お手伝いします」でない）
- [ ] `description` に具体的なキーワードが含まれている
- [ ] 引数がある場合、`argument-hint` フィールドが存在し値がクォートで囲まれている（`"[arg1] [arg2]"` 形式）
- [ ] SKILL.mdボディが500行以下
- [ ] 追加の詳細情報は別ファイルに分離されている（必要な場合）
- [ ] 時間に敏感な情報がない（または「古いパターン」セクションにある）
- [ ] スキル全体で一貫した用語を使用している
- [ ] 例が抽象的でなく具体的
- [ ] ファイル参照はSKILL.mdから1レベル深い（深いネストなし）
- [ ] 段階的開示が適切に使用されている
- [ ] 複雑なワークフローに明確なステップがある
- [ ] Windowsスタイルのパス（`\`）が使われていない（すべてフォワードスラッシュ）

---

## コードとスクリプト（該当する場合）

- [ ] スクリプトが問題を解決し、AIに任せていない
- [ ] エラー処理が明示的で有用なメッセージを含む
- [ ] マジックナンバーがない（すべての値に理由が説明されている）
- [ ] 必要なパッケージがリストされ、インストール方法も明示されている
- [ ] スクリプトに明確なドキュメントがある
- [ ] 重要な操作に検証・確認ステップがある
- [ ] 品質が重要なタスクにフィードバックループが含まれている
- [ ] スクリプトの実行意図が明確（「実行する」vs「参照として読む」）
- [ ] MCPツールがある場合、`ServerName:tool_name` 形式で完全修飾されている

---

## テスト

- [ ] 少なくとも3つの評価が作成されている
- [ ] 対象モデル（Haiku、Sonnet、Opusなど）でテストされている
- [ ] 実際の使用シナリオでテストされている
- [ ] チームフィードバックが組み込まれている（該当する場合）

---

## 採点ガイド

以下の基準で総合評価を決定する：

| 評価 | 基準 |
|------|------|
| **優良** | すべてのコア品質チェック項目が合格。問題点がない。 |
| **良好** | コア品質チェック項目の9割以上が合格。軽微な問題のみ。 |
| **要改善** | コア品質チェック項目で複数の問題。または重大な問題が1つ。 |
| **要大幅改善** | 基本的な構造・説明・簡潔さに根本的な問題がある。 |

**重大な問題の例：**
- `description` が三人称でない（発見に直接影響）
- `description` がいつ使うかを説明していない（スキルがトリガーされない）
- `name` が動名詞形でない（命名規則違反）
- `argument-hint` の値がクォートなし（他のAIで解釈エラーの可能性）
- SKILL.mdが500行を大幅に超えている
- 深くネストされた参照構造（AIが情報を見つけられない）

**軽微な問題の例：**
- 一部の例が抽象的
- 目次がない100行超の参照ファイル
