## 簡潔さが鍵

コンテキストウィンドウはシステムプロンプト・会話履歴・他スキルのメタデータ・実際のリクエストとの共有リソース。SKILL.md内のすべてのトークンが会話履歴と競合する。

スキルのメタデータ（名前と説明）は起動時に常時ロードされる。SKILL.mdボディはスキルが関連するときのみロードされ、追加ファイルは必要に応じてのみロードされる。

**AIがすでに知っていることは書かない。** 各情報に対して問い直す：
- 「AIは本当にこの説明が必要か？」
- 「AIがこれを知っていると仮定できるか？」
- 「このパラグラフはそのトークンコストに見合うか？」

**良い例（簡潔・約50トークン）：**
```markdown
## PDFテキストの抽出

pdfplumberでテキストを抽出する：

```python
import pdfplumber
with pdfplumber.open("file.pdf") as pdf:
    text = pdf.pages[0].extract_text()
```
```

**悪い例（冗長・約150トークン）：**
```markdown
## PDFテキストの抽出

PDF（ポータブルドキュメントフォーマット）ファイルは、テキスト、画像、その他のコンテンツを含む一般的なファイル形式です。PDFからテキストを抽出するには、ライブラリを使用する必要があります。pdfplumberをお勧めします...
```

簡潔なバージョンは、AIが何がPDFか・ライブラリの仕組みを知っていることを前提にしている。

## 適切な自由度の設定

タスクの脆弱性と可変性のレベルに合わせて指示の具体性を調整する。

### 高い自由度（テキストベースの指示）

複数のアプローチが有効・決定がコンテキスト依存・ヒューリスティックがアプローチをガイドする場合に使用。

例：コードレビュープロセス（コンテキストが最適なアプローチを決定する）

### 中程度の自由度（疑似コードまたはパラメータ付きスクリプト）

推奨パターンが存在するが、ある程度の変動が許容される場合に使用。

### 低い自由度（特定のスクリプト、パラメータなし）

操作が脆弱でエラーが発生しやすい・一貫性が重要・特定のシーケンスが必要な場合に使用。

例：データベースマイグレーション（正確なシーケンスが必要）

**アナロジー：**
- **崖のある橋**（低い自由度）：安全な方法は1つ。具体的なガードレールと正確な指示が必要。
- **開けた野原**（高い自由度）：多くのパスが成功につながる。一般的な方向を示し、AIが最適なルートを見つける。

## 使用予定のすべてのモデルでテストする

スキルの効果は基礎となるモデルに依存する。

**モデル別のテスト考慮事項：**
- **Haiku系**（高速・経済的）：スキルは十分なガイダンスを提供しているか？
- **Sonnet系**（バランス型）：スキルは明確で効率的か？
- **Opus系**（強力な推論）：スキルは過度な説明を避けているか？

Opusで完璧に機能するものは、Haikuではより詳細が必要な場合がある。複数のモデルで使用する場合は、すべてで機能する指示を目指す。
