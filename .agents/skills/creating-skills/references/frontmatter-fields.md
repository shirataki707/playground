# SKILL.md フロントマターフィールドリファレンス

## 利用可能なフィールド一覧

| フィールド | 必須 | 説明 |
|---|---|---|
| `name` | Required | スキルの表示名。省略時はディレクトリ名を使用。小文字・数字・ハイフンのみ（最大64文字） |
| `description` | Required | スキルの内容と使いどころ。AI Agentがスキルを適用するか判断に使う。`description` と `when_to_use` の合計は1,536文字で打ち切られるため冒頭に主要ユースケースを書く |
| `when_to_use` | Optional | スキルを起動すべき追加コンテキスト（トリガーフレーズや例など）。`description` に追記される形でカウントされる |
| `argument-hint` | Optional | オートコンプリート時に表示される引数のヒント。例: `"[issue-number]"` や `"[filename] [format]"` |
| `arguments` | Optional | `$name` 置換に使う名前付き位置引数。スペース区切り文字列またはYAMLリストで指定。名前は引数の位置順にマップされる |
| `disable-model-invocation` | Optional | `true` にするとAI Agentが自動ロードしなくなる。`/name` で手動トリガーしたいワークフローに使用。サブエージェントへのプリロードも抑制される。デフォルト: `false` |
| `user-invocable` | Optional | `false` にすると `/` メニューから非表示になる。ユーザーが直接起動すべきでないバックグラウンド知識に使用。デフォルト: `true` |
| `allowed-tools` | Optional | このスキルがアクティブ時に許可なく使えるツール。スペース区切り文字列またはYAMLリストで指定 |
| `model` | Optional | このスキルがアクティブ時に使用するモデル。現在のターンのみ適用され、次のプロンプトでセッションモデルに戻る。**値はスキル選択ダイアログに表示されるモデル名（スラッグ形式）を使用する。不明な場合はユーザーに確認する（`AskUserQuestion`）** |
| `effort` | Optional | このスキルがアクティブ時のエフォートレベル。セッションの設定を上書きする。選択肢: `low` / `medium` / `high` / `xhigh` / `max` |
| `context` | Optional | `fork` を指定するとフォークしたサブエージェントコンテキストで実行する |
| `agent` | Optional | `context: fork` 設定時に使用するサブエージェントの種類 |
| `hooks` | Optional | このスキルのライフサイクルにスコープされたフック |
| `paths` | Optional | スキルが自動起動するファイルパターン（glob）。カンマ区切り文字列またはYAMLリスト。**`*` はカレントディレクトリ直下のみ、`**` はサブディレクトリを含む再帰マッチ。Androidのようなマルチモジュール構成では `**/*.kt` が適切** |
| `shell` | Optional | スキル内の `!command` や `!`-ブロックで使うシェル。`bash`（デフォルト）または `powershell` |

## `paths` の glob 表記例

| パターン | マッチ対象 |
|---|---|
| `**/*.kt` | すべてのサブディレクトリ配下の `.kt` ファイル（マルチモジュール Android プロジェクト推奨） |
| `*.kt` | リポジトリルート直下の `.kt` ファイルのみ |
| `src/**/*.ts` | `src/` 以下のすべての `.ts` ファイル |
| `**/*.{kt,kts}` | すべての `.kt` と `.kts` ファイル |
| `feature/**/*.kt,core/**/*.kt` | `feature/` と `core/` 以下の `.kt` ファイル（カンマ区切りで複数指定） |

> **判断基準**: モジュール分割されたプロジェクトでは原則 `**/*.拡張子` を使用する。ルート直下の単一ファイルのみが対象なら `*.拡張子` を使用する。

## `model` フィールドの補足

`model` フィールドの値はスキルを実行する AI Agent で使用可能なモデルのスラッグを指定する。モデルのスラッグ一覧はセッション設定やスキル選択ダイアログで確認できる。**ユーザーがモデル名を指定していない場合は `AskUserQuestion` で確認するか、フィールド自体を省略する。** モデル名を推定で記述してはならない。

## `description` / `name` のルール

- `name` は動名詞形（動詞 + -ing）必須 → `creating-skills` ✓ / `skill-creator` ✗
- `name` に `anthropic`・`claude` を含まない
- `description` は三人称で書く — 「〜をお手伝いできます」はNG、「〜を実行します」がOK
- `description` に「何をするか」＋「いつ使うか」の両方を含める
- ユーザーがよく使うキーワードを盛り込み、トリガーされやすくする
- `argument-hint` に値がある場合は必ずダブルクォートで囲む

## 引数フィールドの決定ルール

スキル作成時の `argument-hint` / `arguments` は**推測禁止**。必ず以下の順で判断する。

1. **引数なし** → `argument-hint` も `arguments` も追加しない

2. **引数あり** → このファイルの `argument-hint` / `arguments` の正式ルールを確認してから frontmatter を書く

3. **`argument-hint` のルール**
   - 引数がある場合のみ追加する
   - **値は必ずダブルクォートで囲む**
   - 引数の並び順は、本文の説明順と一致させる
   - 例: `argument-hint: "[target-repo] [days]"`

4. **`arguments` のルール**
   - このファイルに正式なYAMLスキーマが書かれている場合のみ追加する
   - 書かれていない項目（`required`・`type` など）を自己判断で足さない
   - フィールド名・入れ子構造・型名・必須属性を推測しない

5. **参照先が不十分な場合のフォールバック**
   - `arguments` は追加しない
   - `argument-hint` のみ使う
   - 本文で各引数の意味・必須/任意・例を明記する
   - 必要なら `AskUserQuestion` で「`arguments` なしで進めるか」を確認する

### 引数があるスキルの本文に含める情報

引数がある場合は、本文に少なくとも以下を含める:

- 各引数が何を表すか
- 必須か任意か
- 1つの具体例
- 引数が省略されたときの扱い（ある場合のみ）

例:

```md
## 引数

- `target-repo`（必須）: 調査対象のリポジトリ名。例: `android`
- `days`（任意）: 対象期間（日数）。省略時は 30 日。例: `30`
```
